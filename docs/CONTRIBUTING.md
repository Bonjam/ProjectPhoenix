# Contributing

Thank you for your interest in Project Phoenix.

Project Phoenix is a lightweight disaster recovery toolkit for self-hosted Docker environments.

The project is still early, but contributions, testing, feedback, documentation improvements, and ideas are welcome.

## Project Goals

Project Phoenix should remain:

- Lightweight
- Simple to understand
- Raspberry Pi friendly
- NAS friendly
- Bash-first
- Recovery focused
- Privacy respecting

## Before Contributing

Please keep these principles in mind:

- Do not add heavy dependencies unless there is a strong reason.
- Do not require a database for core functionality.
- Do not require a web server for core functionality.
- Do not add telemetry or tracking.
- Do not commit secrets, keys, logs, or real config files.
- Keep the CLI usable without Docker.
